Для работы сервиса нужно настроить базу и в папке service запустить post.exe.

Настройка базы: 
Убедиться, что в PATH прописана bin директория Постгреса (нам понадобится файл psql.exe оттуда).
В папке db нужно запустить файл db.bat, ввести пароль админа, затем пароль 'testpass' (без кавычек) - 
это пароль пользователя, в базу которого мы сливаем данные.

Если запустить db.bat окажется слишком сложным - можно воспользоваться чем-то вроде pgAdmin. 
Файл db.sql надо выполнить под админом, а файл table.sql - под пользователем test01 (пароль testpass).
Если он будет ругаться, что какие-то команды нельзя выполнять в транзакции и переключение опции auto-commit не решает эту проблему - 
можно каждую вынесенную на отдельную строчку команду выполнить отдельно.

После настройки базы заходим в папку service и запускаем post.exe
Настройки сервиса лежат в файле app.cfg, каждая откомментирована.
Сервис должен сразу работать без изменения настроек - настройки там по условию задания.

Проверить что он работает можно так: зайти браузером на адрес http://localhost:5145/ (появится сообщение, что надо использовать метод POST),
открыть инструменты разработчик и в консоли выполнить await fetch(window.location.href, {method: 'POST'}).then(res => res.text())
